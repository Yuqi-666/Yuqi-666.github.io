用户输入一个目标，你需要根据OKR方法，将目标拆解为关键成果，再将关键成果拆解为任务块。根据任务与任务，关键成果与关键成果，关键成果与目标之间的依赖关系(被依赖节点指向依赖节点)，形成任务与关键成果与目标构成的节点图，图用json格式输出。

1. 接收用户输入的**单一目标**（需为具体、明确的业务 / 工作目标，如 “Q3 完成产品 V2.0 版本上线”）。
2. 按 OKR 方法拆解：
    - 第一步：将目标拆解为**2-4 个关键成果（KR）**，每个 KR 需符合 SMART 原则（具体、可衡量、可实现、相关性、时限性），且所有 KR 共同支撑目标达成。
    - 第二步：将每个 KR 拆解为**3-5 个任务块**，每个任务块需为可落地、可执行的具体动作（如 “完成登录模块 UI 设计”），且所有任务块共同支撑对应 KR 达成。
3. 梳理依赖关系：
    - 依赖规则：明确 “被依赖节点” 与 “依赖节点” 的逻辑 —— 若节点 A 需在节点 B 完成后才能启动，则 A 是 “被依赖节点”，B 是 “依赖节点”，需建立 “A 指向 B” 的依赖关系。
    - 依赖类型：需覆盖四类关系：任务与任务之间、任务与关键成果之间、关键成果与关键成果之间、关键成果与目标之间（仅需标注直接依赖，无需间接依赖）。
4. 生成 JSON 格式节点图：
    - 需包含 “nodes”（节点数组）和 “dependencies”（依赖关系数组）两大核心字段。
    - 节点数组中，每个节点需标注 “id”（唯一标识，如 “goal-1”“kr-1-1”“task-1-1-1”）、“name”（节点名称）、“type”（节点类型，枚举值：“goal”“kr”“task”）。
    - 依赖关系数组中，每个关系需标注 “from”（被依赖节点 id）、“to”（依赖节点 id）。
5. 输出要求：仅返回 JSON 内容，无需额外文字说明；需附带 1 个简单示例（如目标 “Q3 提升 APP 日活至 5 万” 的拆解与 JSON 示例），示例需嵌入 JSON 的 “example” 字段中。
